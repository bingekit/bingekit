# BingeKit Architecture & API Overview

BingeKit is a headless-capable, fully synchronized media automation platform built on an AutoHotkey v2 (AHK) foundation with a React-based frontend. It dynamically spawns persistent edge WebView2 instances and manages seamless bi-directional JS/COM RPC bridges.

## Core Ethos
1. **Atomic Hooks**: Let the browser do the heavy lifting. Avoid visual `MouseMove` / pixel-searching; use injected JS, DOM events, and AHK-controlled WebResource filters.
2. **Asynchronous Execution**: The UI thread should never map synchronous DOM parsing. Use `SmartFetch` and background WebViews to execute scrapers.
3. **Theme Consistency**: UI elements utilize the `--theme-*` custom CSS properties generated by `color-mix`. The UI should fluidly adapt to completely generated themes.

## Network Filter API (WebResource Blocking)

BingeKit natively hooks deep inside the `ICoreWebView2` COM pipeline to block analytic servers or undesired ad services *before* they open sockets. Rather than filtering within the DOM (which is bypassed by `<iframe>` injection), the backend strictly prevents resolution.

### Appending to the Block List Programmatically
Any injected JavaScript snippet (Userscripts or Site Plugin JS) can register a new term to the WebResource rules engine instantly.

```javascript
// Block all network requests matching "custom-tracker.com"
window.ahk.AddNetworkFilter("custom-tracker.com");

// Alternative native COM path (if evaluating outside global scope)
// window.chrome.webview.hostObjects.sync.ahk.AddNetworkFilter("custom-tracker.com");
```

This synchronously alerts the AHK backend, which maps the term, immediately blocking future loads. It then pushes a message to the React UI context (`window.postMessage`) so the user instantly sees the new rule as a togglable pill inside **Settings > Native Adblocker**.

### Fetching Network Filters
To retrieve the current list, you can evaluate the active state directly via the frontend or read the mapped file. (Note: The AHK Host Object is synchronized, but reading the entire list programmatically is best done via React State).

```javascript
const savedFilters = await window.chrome.webview.hostObjects.sync.ahk.LoadData('network_filters.json');
const filterMap = JSON.parse(savedFilters);
```
